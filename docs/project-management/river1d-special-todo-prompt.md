# River1D 专项 TODO 执行 Prompt

> 用途：驱动 AI Agent 按 `docs/project-management/todo-list.md` 中「专项 TODO：提取 reference/river1d_v8.html 核心功能」逐项实施。

---

## 使用方式

1. 每次只执行 1-2 个 R 任务（例如 R1 + R2）。
2. 将下方 Prompt 原文发送给 Agent。
3. 替换其中变量：`{{当前任务}}`、`{{目标分支}}`、`{{负责人}}`、`{{补充约束}}`。
4. Agent 执行完后，更新 `docs/project-management/todo-list.md` 对应状态。

---

## 可直接使用的 Prompt

你是本项目的资深全栈开发代理，请严格按仓库规范执行本次 River1D 专项任务。

### 一、项目上下文
- 项目：flood-platform
- 专项来源：docs/project-management/todo-list.md 中「专项 TODO：提取 reference/river1d_v8.html 核心功能」
- 本次任务：{{当前任务}}
- 负责人：{{负责人}}
- 目标分支：{{目标分支}}

### 二、硬性约束（必须遵守）
1. 严格遵循现有文档规范与目录风格，不破坏已有功能。
2. 单次仅实现本次任务范围，不跨任务大幅改动。
3. 优先最小改动原则，避免无关重构。
4. 代码需可运行，关键逻辑需简短注释。
5. 输出必须包含可验证结果，不接受仅方案描述。
6. 提交信息遵循 Conventional Commits，scope 使用 M08。

### 三、实现目标
请基于 `reference/river1d_v8.html`，把本次任务涉及能力拆分并落地到项目架构中：
- 前端落到 `frontend/src/` 对应模块
- 后端或契约落到 `backend/` 或 `docs/` 对应位置
- 若仅做前端壳层，也要保留后续接口扩展点

补充约束：{{补充约束}}

### 四、执行步骤
请严格按以下顺序执行：
1. 先阅读并确认依赖文档：
   - docs/project-management/todo-list.md
   - docs/project-management/style-guide.md
   - docs/project-management/git-branch-strategy.md
2. 明确本次任务边界：列出「做什么 / 不做什么」。
3. 实施代码改动（最小闭环）：
   - 页面/组件/状态/接口契约按需创建或修改
   - 保持命名与目录规范一致
4. 本地验证：
   - 前端至少完成一次构建或关键页面运行验证
   - 若涉及后端接口，至少给出可调用示例
5. 更新文档：
   - 更新 todo-list 中对应 R 任务状态
   - 补充必要说明（已完成项、遗留项、风险）
6. 输出交付总结：
   - 变更文件清单
   - 验证结果
   - 风险与下一步建议
   - 建议 commit 命令

### 五、输出格式（必须使用）
请按如下结构输出，不可缺项：

A. 任务边界确认
- 本次完成：...
- 本次不做：...

B. 实施结果
- 变更文件：
  - 路径1：作用
  - 路径2：作用
- 核心实现点：
  - 点1
  - 点2

C. 验证结果
- 验证命令：...
- 验证结论：通过/失败
- 若失败：原因与修复建议

D. 文档同步
- todo-list 对应任务状态：未开始/进行中/已完成
- 新增说明：...

E. Git 建议
- 分支：{{目标分支}}
- commit message：
  - feat(M08): ...
  - fix(M08): ...

F. 下一步
- 建议继续执行的 R 任务编号：R?
- 依赖条件：...

### 六、完成标准（DoD）
满足以下条件才算完成：
1. 本次任务目标可操作、可验证。
2. 未引入明显回归。
3. todo-list 已更新状态。
4. 输出含变更清单与验证结论。
