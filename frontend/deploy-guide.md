# 前端部署指南

## 方案B1：打包 + 简易HTTP服务器

### 1. 构建生产版本
```bash
cd frontend
npm run build
# 生成 dist/ 目录
```

### 2. 使用 Node.js 启动静态服务器
```bash
# 安装 serve（如果没有）
npm install -g serve

# 启动服务器
cd dist
serve -s . -l 8080
```

### 3. 在另一台电脑访问
```
http://你的IP:8080
```

---

## 方案B2：打包 + 复制到U盘

### 1. 构建 + 打包静态服务器
```bash
cd frontend
npm run build

# 将 dist/ 目录复制到 U盘
# 下载便携版 serve.exe（或使用 Python）
```

### 2. 在领导电脑上
```bash
# 方式1：使用 Python（Win10/11自带）
cd dist目录
python -m http.server 8080

# 方式2：使用 serve.exe
serve.exe -s . -l 8080
```

### 3. 浏览器访问
```
http://localhost:8080
```

---

## 方案C：完整项目复制

### 前置条件
领导电脑需要安装：
- Node.js 18+
- Git（可选）

### 步骤
1. 复制整个 `flood-platform` 文件夹到U盘
2. 在领导电脑：
   ```bash
   cd flood-platform/frontend
   npm install
   npm run dev
   ```
3. 访问 http://localhost:3002

## 🎯 推荐方案总结

| 场景 | 推荐方案 |
|------|---------|
| 同一办公室，快速演示 | 方案A（局域网） |
| 正式汇报，需要流畅 | 方案B1（打包+serve） |
| 异地演示，无网络 | 方案B2（U盘+Python） |
| 领导要自己操作 | 方案C（完整复制） |

## ⚠️ 注意事项

1. **浏览器兼容**：推荐 Chrome/Edge（避免 IE）
2. **防火墙**：局域网访问可能被防火墙拦截，需允许端口
3. **端口占用**：如果3002被占用，Vite会自动换端口
4. **Mock数据**：当前是Demo版本，数据为模拟数据
5. **性能**：大屏建议全屏展示（F11），分辨率1920×1080最佳
